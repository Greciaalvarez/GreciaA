Automated Trading System (ATS)

Quantitative Engineering & Algorithmic Strategy | Forex Market

1. Research Premise: Context vs. Permission

Este sistema se basa en una investigación cuantitativa sobre el análisis de múltiples marcos temporales. Basándome en la teoría de Brian Shannon (2008), el sistema diferencia entre el Contexto de Mercado (H4) y el Permiso de Entrada (H1). Evitar la sobre-optimización (over-fitting) al buscar alineación perfecta en todos los marcos temporales fue clave para mantener la robustez del sistema.

2. Technical Architecture (MQL5)

La transición de un prototipo en Python a un motor de ejecución en MQL5 permitió transformar lógica discrecional en un entorno de ejecución algorítmica 24/7. El sistema utiliza una arquitectura basada en eventos (event-driven) que procesa señales en tiempo real.

Highlights Técnicos:

  • Bias Directional (H4): Utiliza cruces de EMAs (20/50) para establecer la dirección institucional.
  • Gestión de Volatilidad (ATR): Implementación de stop-loss dinámico basado en un multiplicador (1.72) del Average True Range.
  • Risk Governance: Cálculo de lotaje automático ajustado al capital (Equity) en tiempo real.

3. Execution Methodology

El siguiente fragmento de código ilustra la lógica central de protección de riesgo y dimensionamiento de posición antes de enviar la orden al servidor del broker:

// ATR-Based Stop Loss & Position Sizing Logic double sl = structuralLow - (atr * 1.72); double lot = CalculateLotSize(MathAbs(close1 - sl)/_Point); double tp = close1 + (MathAbs(close1 - sl) * 2); // Execute trade if all quantitative thresholds are met OpenTradeBuy(lot, sl, tp);

4. Conclusión

La investigación demuestra que la automatización no solo elimina el sesgo emocional, sino que permite una ejecución consistente de reglas estadísticas. Este sistema no busca la "operación perfecta", sino la **disciplina algorítmica** necesaria para operar en mercados complejos con una gestión de riesgo cuantificada.
